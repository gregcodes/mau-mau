Tox - a developer frontendÂ¶

Tox is one abstraction level up from py.test and serves as a command line frontend for different kinds of tests and also acts a a frontend for external test runners as part of CI. It automatically creates an environment for the tests, installs dependencies, does whatever else is needed for test preparation and outputs the results.

It can also be (ab)used to act as a frontend for other tasks developers have to do as part of their work. In this project we also use it for building and developing the documentation.

Run static code analysis:

$ cd </path/to/your/clone>
$ tox -e static
